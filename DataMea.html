Project Name: DataMea
Client: SUNY Oswego
Year: 2018
Go to project: DataMea

 

Project Description:
DataMea is a desktop analytics application developed at SUNY Oswego as part of a software design course. DataMea analyzes the userâ€™s Internet Message Access Protocol (IMAP) data to display detailed, and sophisticated visualizations of email activities. The visualizations are displayed in a dashboard format that includes, gauges, heat maps, detailed bar and pie graphs, sunburst charts, tree maps and more. The application provides users with interactive capabilities, enabling them to manipulate the charts or graphs.

Product Functions
Analyze Email: the action of signing in to one's email via DataMea to then have the application analyze the IMAP data which will display on the product's dashboard automatically.

Exit: the action in which the application closes out of itself safely and ceases running on the user's device.

Store Data: DataMea will store basic incremental/decremental data in a read/write file on the user's local storage where the software is installed. No personal email data will be logged/saved besides the sender and receiver email usernames which will be encrypted along with the entirety of the file(s).

Process Request Waiting Screen: Once the IMAP data analysis is being processed, a separate window will display with a brief tutorial on how to use the software and a progress bar.

User Email Alert: A method in which is tied to its correlating check box to alert the user via email when the analysis is complete.

Engine Filter: Once the user refines the applicable data, DataMea will adjust the corresponding graphs accordingly.
